# Phase 5 共通: ユーザ動作確認・文書反映

## 実施内容

1. ユーザ向けの動作確認観点を整理して共有する
2. アプリ起動での動作確認を依頼し、結果報告を待つ

> ⛔ **STOP**: ユーザの動作確認結果が返るまで、次の工程に進んではならない。

3. NG の場合は原因分析を記録して Phase 4 へ戻る
4. OK の場合は workflow 固有の反映先へ文書を更新する
5. Phase 5 の文書反映成果をコミットする
6. ユーザへドキュメントレビュー依頼準備完了を報告し、レビュー完了まで待機する

> ⛔ **STOP**: ユーザにドキュメントレビュー依頼の準備完了を報告する。ユーザがレビュー担当 Agent を起動し、ドキュメントレビューが完了するまで待機する。

> 補足: `claude-review-automation` を使う場合、準備完了の報告後はユーザの追加承認を待たずに Codex が Claude review を起動してよい。監視は `15s -> 30s -> 60s` の backoff を使い、5 分程度の `thinking` は通常範囲として扱う。10 分程度進捗が見えない場合のみ、再起動前にユーザへ対応方針を確認する。Phase 区切りでは短い状態要約を残し、利用可能なら `/compact` を実行してよい。

7. レビュー指摘を反映し、文書更新をコミットする

> ⛔ **STOP**: Phase 5 完了をユーザに報告し、Phase 6 への進行承認を待つ。承認なしに Phase 6 に進んではならない。

## レビュー担当への依頼

- レビュー担当 Agent は `docs/procedure/ai_review_response_workflow.md` に従う
- レビュー観点詳細は `docs/procedure/review_checkpoints.md` を参照する

## 完了条件

- ユーザ動作確認の結果が記録されている
- 文書反映に対するレビュー指摘が全て対応済みである
- Phase 5 成果がコミットされている

## 承認ゲート

- ドキュメントレビュー完了後、ユーザ承認なしに Phase 6 へ進まない
