# issue-resolution Phase 2: 方針・設計の観点

## 設計書

- `docs/design_analysis/issue_resolution/<yyyymmdd>_<topic>/design/<topic>_issue_resolution_design.md`

## 必須観点

- 現状課題の原因仮説または欠落整理
- 解決後の完了条件
- 測定方法または確認証跡
- 解決対象の課題を閉じるための設計変更
- 影響範囲と切り戻しポイント
- 効果確認方法
- 類似ロジックの抽象化・共通化方針
- 明示要求または外部契約がない限り、不要な経路を残さない方針
- プロジェクト内で閉じる仕様不一致に対する例外・エラーハンドリング方針
- 恒久ドキュメント更新予定先
- スコープ外へ切り出す follow-up
- 残課題の扱い

## レビュー観点

- 課題を「終わった」と言える条件が明確か
- 設計が完了条件に直接つながっているか
- 解決方針が大きすぎず閉じた範囲になっているか
- 別課題を無制限に抱き込んでいないか
- 効果確認が設計段階で準備されているか
