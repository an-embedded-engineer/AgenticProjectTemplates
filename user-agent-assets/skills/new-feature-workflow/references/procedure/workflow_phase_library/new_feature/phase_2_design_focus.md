# new-feature Phase 2: 方針・設計の観点

## 設計書

- `docs/design_analysis/new_feature/<yyyymmdd>_<topic>/design/<topic>_feature_design.md`

## 必須観点

- ユースケースと操作導線
- 最小提供範囲、非対象、後続拡張
- 既存機能との統合方針
- 機能追加に必要な責務分割
- UI / データ / サービス / 永続化の追加点
- 段階導入、初期値、migration の要否
- 拡張ポイントと将来の派生機能への耐性
- 失敗時動作とデフォルト挙動
- 既存類似ロジックとの抽象化・共通化方針
- 不要な代替経路を並存させない単一路線の設計方針
- 恒久ドキュメント更新予定先
- テスト観点とユーザ検証観点
- リスクと follow-up

## レビュー観点

- 新機能の価値が受け入れ条件に落ちているか
- 境界条件と導線が明確か
- 既存仕様との衝突が整理されているか
- 既存アーキテクチャへ無理なく収まるか
- 類似責務の重複追加や不要な互換レイヤーに依存していないか
- 将来拡張の余地を潰していないか
