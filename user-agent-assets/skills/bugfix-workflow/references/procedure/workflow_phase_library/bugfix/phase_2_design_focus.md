# bugfix Phase 2: 方針・設計の観点

## 設計書

- `docs/design_analysis/fix_issues/<yyyymmdd>_<topic>/design/<topic>_bugfix_design.md`

## 必須観点

- 症状、再現条件、期待値 / 実際値
- 根本原因と誘発条件の切り分け
- 恒久修正方針
- 根本原因に対する修正ポイント
- 根本原因を解消する必要十分な差分での修正方針
- 重複ロジックや責務分散が原因の場合は、抽象化・共通化で解消する方針
- 例外処理とログ方針
- 後方互換レイヤーや不要フォールバックを増やさない方針
- 回帰リスク
- 回帰テストの設計
- 恒久ドキュメント更新予定先
- 一時回避策が必要な場合の扱い

## レビュー観点

- 症状対策ではなく原因対策になっているか
- 原因と修正点の対応が一対一で追えるか
- 副作用のある広範囲修正になっていないか
- 条件分岐の継ぎ足しより、単一路線化や共通化に寄せられているか
- 不必要なフォールバックで不具合を隠蔽していないか
- 回帰防止が設計に含まれているか
