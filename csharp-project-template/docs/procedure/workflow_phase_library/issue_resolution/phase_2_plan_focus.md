# issue-resolution Phase 2: 計画書の観点

## 計画書

- `docs/design_analysis/issue_resolution/<yyyymmdd>_<topic>/plan/<topic>_issue_resolution_plan.md`

## 必須観点

- 現状課題の原因仮説または欠落整理
- 解決後の完了条件
- 測定方法または確認証跡
- スコープ外へ切り出す follow-up
- テストまたは確認項目

## レビュー観点

- 課題を「終わった」と言える条件が明確か
- 解決方針が大きすぎず閉じた範囲になっているか
- 効果測定の方法が曖昧でないか
