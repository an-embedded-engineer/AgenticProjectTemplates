# bugfix Phase 3: 設計書の観点

## 設計書

- `docs/design_analysis/fix_issues/<yyyymmdd>_<topic>/design/<topic>_bugfix_design.md`

## 必須観点

- 根本原因に対する修正ポイント
- 根本原因を解消する必要十分な差分での修正方針
- 重複ロジックや責務分散が原因の場合は、抽象化・共通化で解消する方針
- 新規メソッド/関数の配置先を責務で判断し、呼び出し箇所に近いだけの場当たり配置を避ける方針
- トップレベル関数相当や責務の曖昧な `static` helper を安易に増やさず、責務に応じた型/メソッドへ収める方針
- 例外処理とログ方針
- 後方互換レイヤーや不要フォールバックを増やさない方針
- 回帰テストの設計

## レビュー観点

- 原因と修正点の対応が一対一で追えるか
- 副作用のある広範囲修正になっていないか
- 条件分岐の継ぎ足しより、単一路線化や共通化に寄せられているか
- メソッド/関数配置が責務に沿っており、場当たり的な helper 追加で問題を隠していないか
- 失敗時に追跡可能な振る舞いが確保されているか
