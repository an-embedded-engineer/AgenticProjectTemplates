# レビュー観点（詳細）

## 1. 仕様整合

- ドキュメント仕様と実装挙動に差分がないか
- 仕様未記載の制約がコードに埋め込まれていないか

## 2. 設計品質

- 責務分離（SRP）が保たれているか
- 既存設計パターンに矛盾しないか
- 不要な重複実装（DRY違反）がないか
- API/命名/責務分離の一貫性が保たれているか

## 3. 安全性・保守性

- 例外を握りつぶしていないか
- 不要なフォールバックで不具合を隠していないか
- ログに原因追跡情報が残るか

## 4. パフォーマンス

- UI スレッドをブロックする I/O がないか
- パフォーマンス制約（タイムバジェット等）を考慮しているか
- 不要な再計算やメモリ確保がないか

## 5. 互換性・回帰

- 既存 API/UI の期待動作を壊していないか
- 既存テストケースの意味が変わっていないか

## 6. テスト

- 変更点を直接検証するテストがあるか
- 境界値・失敗系が網羅されているか
- 統合テスト条件を満たしているか

## 7. ドキュメント

- 設計書とレビュー書の対応ステータスが一致しているか
- 関連コンポーネント文書へ反映済みか
- 記載不備/不整合を見つけた場合にその場で修正したか
